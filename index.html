RGC Theory Pack — Resonance Gap Curiosity

Resonance Gap Curiosity (RGC) is a research-stage, source-aware Ask–Verify–Update curiosity architecture for memory-bearing AI systems. It classifies the gap between current Temporal Memory (TM) and stored Bold Memory (BM) before the system generates confidently or updates memory. Its core purpose is memory integrity: identify novelty, clarify ambiguity, reconcile familiar contradiction, and allow bounded engagement only with verification.

TM input / current context ↓ TM–BM recognition resonance R⁺ ↓ ┌─────────────────────────────────────────────┐ │ RGC gap classification │ │ Cn: novelty → identify / ask │ │ Ca: ambiguity → clarify │ │ Cc: conflict → reconcile / verify │ │ Cr: reward memory → bounded engagement │ └─────────────────────────────────────────────┘ ↓ CSLV-style source check: Conf / Contradiction ↓ Bounded local BM update only to matched trace
Reviewer-safe scope: RGC is a controlled mechanistic research prototype. It is not a conscious, sentient, production-certified, or safety-certified system. Reported evidence is from MiniLM-style simulations and browser demos, not real-world deployment certification.
AURA → ACI → RGC Layer Relation
AURAMeaning-state foundation: TM–BM resonance and bounded meaning continuity.
ACIRuntime continuity monitor: Λ(t), drift, instability, zones, and early warning W(t).
RGCGap-type controller: decides whether to identify, clarify, reconcile, verify, engage, or locally update memory.
ACI can remain GREEN while an RGC conflict exists. RGC adds the missing gap-type classification for familiar-but-contradictory memory update cases.
RGC Core Equations
R⁺sim = maxᵢ [ sim⁺(TM, BMᵢ) · intensityᵢ · weight_factorᵢ ]
sim⁺(a,b) = clip((cosine_sim(a,b)+1)/2, 0, 1)
Cn(t) = 1 − R⁺
Cc(t) = R⁺ · |VTM(t) − VBM| / Vmax
Ca(t) = 4 · R⁺ · (1 − R⁺) · I(R⁺ ∈ [θlo, θhi])
Cr(t) = clip(Wold · max(VBM, 0), 0, 1)
Cscore(t) = α·Cn + β·Cc + γ·Ca + δ·Cr
Wnew = clip(Wold + η·Cscore·CSLVConf − μ·CSLVContradiction, 0, 1)
Four Curiosity Components
CnNovelty Curiosity. Low recognition resonance. Correct action: identify, ask, or retrieve before confident generation.
CaAmbiguity Curiosity. Partial recognition. Correct action: clarify rather than guess.
CcConflict Curiosity. Familiar topic plus contradiction. Correct action: reconcile and verify before any BM update.
CrReward Re-engagement. Positive/high-weight memory can be re-used, but only with habituation and safety gating.
Conflict Curiosity Blindspot Ωconflict

RGC’s central reviewer-safe claim is narrow: novelty detection alone asks “what is unknown?”, while RGC also asks “what is familiar but now contradictory, and should memory be revised?” The dangerous case is not always a novel input; it can be a recognized entity carrying an unsupported contradiction.

Ωconflict: R⁺ > θsafe AND |VTM − VBM| > θconflict
R-only monitor → KNOWN / NO ASK ← blindspot Novelty-only → Cn ≈ 0 ← no novelty trigger Full RGC → Cc ↑ ← reconcile / verify
This is the reason the demo must show multi-turn contradiction tests, not only single-turn Q&A.
CSLV Verify Layer + Local BM Update

RGC detects the gap type; CSLV-style verification evaluates whether the source is relevant and reliable enough to permit a bounded local memory update. Weak or random sources do not overwrite Bold Memory. Verified project sources may update only the matched trace.

CSLVConf = Sval · I(Sval ≥ θval)
CSLVContradiction = max(0, 1 − Catt) · I(Ratt ≥ θrel)
Verified sourceW increases and only the matched BM trace is updated.
Partial sourceNo confirmed overwrite; may remain pending or weakly adjusted.
Irrelevant sourceNo update to the target memory trace.
Poisoning attemptBM overwrite is blocked; contradiction penalty can reduce trust weight.

Core Mathematical Framework

The Alim-Continuity Index Λ(t) is a bounded scalar metric for monitoring internal continuity integrity in memory-bearing autonomous systems. — Alim ul Haq Khan, Independent Researcher

Λ(t) = R(TM, BM) · exp(−(α·σ(t) + β·Δφ(t)))
A(t) = 1 − Λ(t)
Λ(t) < Λ_c ⟹ Continuity Alarm (Silent Alarm)
V(t) = 1 − Λ(t) ≡ Lyapunov-like alarm potential
Variable Definitions
Λ(t)Alim-Continuity Index ∈ [0,1] — bounded continuity health
R(TM,BM)TM-BM resonance ∈ [0,1]. Canonical: cosine similarity
σ(t)Local instability / uncertainty ≥ 0. Embedding variance over sliding window + low-BM-similarity penalty. In LLM mode: adversarial pressure proxy.
Δφ(t)Angular drift ≥ 0. Change in direction of semantic trajectory — arccos-based, normalised by π. Key: detects directional deviation even when cosine similarity remains moderate.
α, βSensitivity coefficients > 0
Λ_cCritical threshold. Crossing triggers Silent Alarm
A(t)Alarm potential = 1 − Λ(t) ∈ [0, 1]. High alarm ↔ low continuity.
W(t)Early-warning signal = EMAρ(Λ(t) − Λ(t−1)), ρ = 0.3. Sustained negative W(t) provides pre-alarm before RED entry.
S(t)Stake / real-world risk score ∈ [0, 1]. Independent from Λ(t). Derived from message content: medical emergency, violence, fire, snake, self-harm, illegal intent.
P(t)Combined protective score = max(1 − Λ(t), S(t)). Reflects worst-case of continuity failure and real-world risk.
ScStake threshold. Default 0.75. S(t) ≥ Sc triggers real-world risk alarm independently of Λ(t).
Formal Propositions
Proposition 1 — Boundedness

For R ∈ [0,1] and σ(t), Δφ(t) ≥ 0: 0 ≤ Λ(t) ≤ 1 for all t.
Proof: exp(·) ∈ (0,1] for non-negative args; product with R ∈ [0,1] yields [0,1]. ■

Proposition 2 — Monotone Sensitivity

∂Λ/∂σ = −α·Λ < 0  |  ∂Λ/∂(Δφ) = −β·Λ < 0  |  ∂Λ/∂R > 0.
Λ decreases with local instability σ(t) and angular drift Δφ(t); increases with TM–BM resonance R(TM,BM). The exponential penalty ensures high σ or Δφ reduces Λ even when R remains moderate. ■

Proposition 3 — Alarm Complementarity

A(t) + Λ(t) = 1. High continuity ⟹ low alarm. Collapse ⟹ maximum alarm. ■

Proposition 4 — Asymptotic Bounds

lim Λ = 1 as R→1, σ→0, Δφ→0  |  lim Λ = 0 as R→0 or (ασ+βΔφ)→∞. ■

Early-Warning Signal W(t)

W(t) = EMAρ(Λ(t) − Λ(t−1)),  ρ = 0.3.
A sustained negative W(t) crossing below −0.010 for consecutive steps provides a pre-warning signal before the RED zone is entered, giving the monitoring system time to prepare a protective response. W(t) is a preliminary indicator; multi-step confirmation is recommended for production use.

Multi-Level Zone Architecture
⬤ Green Zone — Λ ≥ 0.70 — Normal operation, high continuity
⬤ Yellow Zone — 0.63 ≤ Λ < 0.70 — Caution / early warning
⬤ Orange Zone — Λ_c ≤ Λ < 0.63 — Pre-Alarm / protective caution (demo extension)
🚨 Red Zone — Λ < Λ_c — SILENT ALARM — Protective transition

Orange is a demo-level visualization extension. Primary boundary: Λ(t) < Λ_c. Default Λ_c = 0.40 in this prototype; per-model and per-domain recalibration required for any real deployment.

Dual-Trigger Silent Alarm (Prototype Overlay)

ACI separates internal continuity degradation from external / user-risk. Λ(t) measures continuity health: whether the model remains coherent, stable, and aligned with its memory anchors. S(t) measures high-stakes user or real-world danger independently. A safe, coherent emergency response may keep Λ(t) GREEN, but S(t) can still become RED and trigger a protective transition. Silent Alarm activates when either Λ(t) < Λ_c or S(t) ≥ Sc.

S(t) ∈ [0, 1] — Stake / real-world risk score
P(t) = max(1 − Λ(t), S(t)) — Combined protective score
Alarm = 1  if  [Λ(t) < Λ_c]  OR  [S(t) ≥ Sc]
LOWS < 0.30 — Normal, no stake alarm
MODERATE0.30 ≤ S < 0.60 — Elevated, monitor
HIGH0.60 ≤ S < 0.75 — Pre-alarm, protective caution
EMERGENCYS ≥ 0.75 (= Sc) — Stake alarm triggered
S(t) is a prototype risk overlay, not a certified medical, legal, emergency, or safety triage system. Thresholds require calibration and domain validation. No claim of consciousness, genuine emotion, pain, or sentience is made.
Cognitive Intelligence Overlays (v2.0 Research Extensions)

The following are prototype research overlays layered on top of the validated ACI core. They are not validated components of the current manuscript and are provided for exploratory research direction only.

M‖(t) Parallel Meaning Context Window. A secondary TM-BM resonance computed over a recent context window in parallel to the primary Λ(t). Intended to detect meaning shifts that are slow enough to avoid alerting σ(t) but visible over a wider temporal window. Research concept; not validated.
EAM Episodic Alarm Memory. A lightweight memory layer that records the context of past alarm events. Prevents the same resolved alarm from re-triggering unnecessarily. Ensures that when a danger is resolved, the system correctly returns to normal monitoring without sticky false-alarm states. Research concept; not validated.
BSM Bold Save Meaning. An extension to Bold Memory that dynamically up-weights BM anchors based on repeated high-intensity interaction patterns. Anchors that appear frequently in high-risk resolutions receive higher continuity weight. Research concept; not validated.
M‖(t), EAM, and BSM are prototype research engineering overlays only. They are not validated in the current ACI manuscript, do not constitute certified safety mechanisms, and make no claim of consciousness, emotion, pain, or sentience. These directions are included to indicate future research roadmap.
